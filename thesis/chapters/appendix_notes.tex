\chapter{Appendix Notes}

\section{Repository Mapping}

The practical implementation described in this thesis is primarily located in the following repository components:

\begin{itemize}
    \item \texttt{uncertainty\_lab/pipeline/} for end-to-end orchestration,
    \item \texttt{uncertainty\_lab/uncertainty/} for uncertainty estimation logic,
    \item \texttt{uncertainty\_lab/metrics/} for predictive, calibration, and selective-prediction metrics,
    \item \texttt{uncertainty\_lab/data/} for dataset loading, and
    \item \texttt{experiments/} for supplementary evaluation scripts.
\end{itemize}

\section{Reproducibility Note}

The compact thesis benchmark introduced in \Cref{chap:results} is linked to dedicated thesis configuration files. The training configuration is stored in \texttt{thesis/configs/train\_small\_vit.yaml}, while the evaluation benchmark is stored in \texttt{thesis/configs/benchmark\_trained\_checkpoint.yaml}. These files can be transferred into a university thesis repository or an Overleaf appendix as implementation evidence.
